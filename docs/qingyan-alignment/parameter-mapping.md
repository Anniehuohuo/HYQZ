# 参数映射表（清言平台 vs 本系统调用）

| 维度 | 清言平台（Console/Web） | 清言 Assistant API（本系统直连） | 本系统当前实现 | 对齐动作（可量化） |
|---|---|---|---|---|
| temperature | 平台可配置（需采集） | v1 文档未体现可在请求覆盖 | 未传，完全依赖平台侧默认 | 平台侧固定到 0~0.2；评测平均 BLEU/ROUGE-L ≥0.85 |
| top_p | 平台可配置（需采集） | v1 文档未体现可在请求覆盖 | 未传，完全依赖平台侧默认 | 平台侧固定为 1（或平台稳定值） |
| max_tokens | 平台可配置（需采集） | v1 文档未体现可在请求覆盖 | 未传，完全依赖平台侧默认 | 平台侧固定值；完整度差异 <5% |
| system prompt | 平台智能体配置 | 请求不传 system，由平台侧应用 | 本系统不注入 system，只传 prompt | 平台侧 prompt 明确“输出格式/引用规则/结构”；小程序端保留结构渲染 |
| 工具/插件开关 | 平台智能体配置 | 取决于平台对 API 的支持 | 本系统无控制能力 | 平台侧导出开关清单；固定评测期间开关不变 |
| 知识库绑定/版本 | 平台智能体配置 | 取决于平台对 API 的支持 | 本系统无控制能力 | 记录知识库版本/更新时间；更新后重新索引并发布；评测期冻结版本 |
| 上下文长度限制 | 平台设置（需采集） | conversation_id 续聊由平台控制 | 本系统仅传本轮 prompt；续聊依赖 conversation_id | 确保同一 session 的 remote conversation_id 稳定复用；评测单轮对齐时不传 conversation_id |
| conversation_id | 平台会话维度 | 支持 | 本系统持久化并复用 | 用同一会话对齐“多轮深度”；评测单轮对齐“不传”以避免跨轮干扰 |
| endpoint / 版本 | 平台可能使用更高版本 | 当前为 `assistant-api/v1` | 固定 v1（现已支持配置） | 对齐为与平台一致的 baseUrl（`qingyan_base_url`）；记录 v1/v2 差异 |
| 响应结构 | 平台富内容（Markdown/卡片/工具结果） | v1 返回 content 可能为多 type | 只抽取 text/markdown（已增强） | 如需更丰富：扩展抽取/映射策略并定义前端展示协议 |
| 前端渲染 | 平台完整 Markdown 渲染 | - | 小程序 rich-text + Markdown 子集 | 保留标题/列表/代码块；渲染一致性通过截图/DOM 用例验收 |
