## 三、内容差异高发根因

- [ ] 端点重试 + 请求体重试命中“可用但非预期路径”，导致回复风格波动
- [ ] 未固定 `temperature/top_p/max_tokens`，平台默认值覆盖
- [ ] `conversation_id` 丢失或跨智能体串用，导致上下文污染
- [ ] SSE 分片含元数据，解析不严导致 `msg_type/ori_req` 混入正文
- [ ] 小程序渲染层仅按文本拼接，结构化字段未利用

## 四、量化评估指标（每次改动都要跑）

- [ ] 文本长度差异率（API vs 扣子调试页）≤ 20%
- [ ] 结构完整度（是否含“评分/亮点/建议”等关键段）≥ 90%
- [ ] JSON 污染率（包含 `msg_type|ori_req|from_module`）= 0
- [ ] 会话连续率（第二轮可引用第一轮语义）≥ 95%
- [ ] 多智能体串话率 = 0

## 五、回归测试用例（最少）

- [ ] 用例A：评分类场景（应输出评分+建议）
- [ ] 用例B：普通问答场景（不应强制评分）
- [ ] 用例C：多轮追问（验证 conversation_id 连续）
- [ ] 用例D：切换智能体后提问（验证隔离）
