# 第一阶段中台托管上线说明

## 目标
- 在不影响原有功能的前提下，新增“中台托管”智能体模式。
- 保留原有“本地/智谱”“清言”两种来源能力。
- 新模式不依赖外部 assistant_id，支持后台直接配置人设与知识内容。

## 兼容性结论
- 原有清言智能体调用链路不变。
- 原有本地/智谱智能体调用链路不变。
- 仅当“来源=中台托管”时，才进入新增逻辑。

## 代码改动清单
- 后端：
  - `crmeb/app/services/ai/AiAgentServices.php`
    - provider 新增 `managed`
    - 新增 provider_meta 字段：`managed_model`、`managed_knowledge`
  - `crmeb/app/adminapi/controller/v1/ai/AiAgent.php`
    - 新增入参：`managed_model`、`managed_knowledge`
  - `crmeb/app/adminapi/validate/ai/AiAgentValidate.php`
    - provider 校验扩展为 `local/qingyan/managed`
    - 新增 `managed_model`、`managed_knowledge` 校验
  - `crmeb/app/services/ai/AiChatServices.php`
    - 新增 `managed` 分支调用
    - 复用现有智谱大模型接口
    - 新增 `buildManagedMessages` 与 `streamOpenModelChat`
- 后台前端：
  - `template/admin/src/pages/ai/agentMatrix/index.vue`
    - 来源新增“中台托管”
    - 新增表单字段：托管模型、知识库内容
    - 列表来源显示新增“中台托管”

## 数据库变更
- 第一阶段无需新增表结构。
- 继续复用 `eb_ai_agents.provider_meta` 存储托管配置。

## 运营使用方式
- 在“智能体矩阵”新增智能体时，选择“来源=中台托管”。
- 填写：
  - `system_prompt`：人设与回复规则
  - `托管模型`：可留空（默认 `glm-4-flash`）
  - `知识库内容`：先填核心知识文本
- 发布后即可在小程序中对话。

## 已知边界
- 第一阶段知识库为“文本托管”，未接入文件上传/切片/向量检索。
- 后续第二阶段可升级到文档型知识库。

## 回滚方案
- 如需回滚，仅需在后台将智能体来源改回“本地/智谱”或“清言”。
- 新增字段保存在 provider_meta，不影响历史智能体运行。
