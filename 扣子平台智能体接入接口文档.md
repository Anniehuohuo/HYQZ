## 1. 概述

### 1.1 文档说明

本文档旨在帮助外部系统开发者了解如何接入扣子（Coze）平台的智能体（Bot），通过 API 调用实现与智能体的交互。

### 1.2 适用场景

- 企业系统集成扣子智能体能力
- 第三方应用调用扣子智能体进行对话
- 自定义业务系统接入 AI 对话功能
- 工作流自动化调用智能体

### 1.3 基础信息

| 项目 | 说明 |
|------|------|
| 基础 URL（国内版） | `https://api.coze.cn` |
| 基础 URL（国际版） | `https://api.coze.com` |
| 认证方式 | Bearer Token |
| 数据格式 | JSON |
| 字符编码 | UTF-8 |

## 2. 快速开始

### 2.1 接入流程

```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   1. 创建智能体  │ -> │  2. 获取 API 凭证  │ -> │  3. 调用 API    │
│   在 Coze 平台    │    │   Personal Access │    │   发送 HTTP 请求  │
│   创建并配置 Bot │    │   Token          │    │   进行对话      │
└─────────────────┘    └─────────────────┘    └─────────────────┘
         │                      │                      │
         v                      v                      v
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   4. 发布智能体  │ <- │  5. 获取 Bot ID  │ <- │  4. 解析响应   │
│   发布为 API 服务 │    │   用于 API 调用   │    │   处理返回结果  │
└─────────────────┘    └─────────────────┘    └─────────────────┘
```

### 2.2 前置准备

1. **注册扣子账号**
   - 访问 https://www.coze.cn 注册账号
   - 完成账号验证

2. **创建智能体**
   - 登录扣子控制台
   - 创建新的 Bot 并配置人设、插件等

3. **发布智能体**
   - 进入 Bot 开发页面
   - 点击"发布"按钮
   - 选择"Bot as API"发布方式
   - 选择发布范围（公开/私有）

## 3. 认证方式

### 3.1 Personal Access Token (PAT)

扣子平台使用 Personal Access Token 进行 API 认证。

#### 3.1.1 获取 PAT

1. 登录扣子控制台：https://console.coze.cn
2. 进入"开发设置"或"API 管理"
3. 点击"创建令牌"
4. 填写令牌名称和权限范围
5. 复制并安全保存生成的 Token（仅显示一次）

#### 3.1.2 使用 PAT

在 HTTP 请求头中添加认证信息：

```http
Authorization: Bearer YOUR_PERSONAL_ACCESS_TOKEN
Content-Type: application/json
```

### 3.2 安全建议

| 建议 | 说明 |
|------|------|
| 安全存储 | 将 Token 存储在环境变量或密钥管理系统中 |
| 定期轮换 | 定期更换 Token，降低泄露风险 |
| 最小权限 | 仅申请必要的权限范围 |
| 禁止硬编码 | 不要将 Token 硬编码在代码中 |

## 4. API 接口详解

### 4.1 对话接口（Chat）

用于与智能体进行单轮或多轮对话。

#### 4.1.1 接口地址

```
POST /v1/chat
```

完整 URL：
```
https://api.coze.cn/v1/chat
```

#### 4.1.2 请求头

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| Authorization | string | 是 | Bearer Token 认证 |
| Content-Type | string | 是 | application/json |

#### 4.1.3 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| bot_id | string | 是 | 智能体 ID |
| user_id | string | 是 | 用户唯一标识 |
| query | string | 是 | 用户输入的问题或消息 |
| conversation_id | string | 否 | 会话 ID，用于多轮对话 |
| stream | boolean | 否 | 是否使用流式响应，默认 false |
| chat_settings | object | 否 | 会话配置参数 |

#### 4.1.4 请求示例

```bash
curl -X POST 'https://api.coze.cn/v1/chat' \
  -H 'Authorization: Bearer YOUR_PAT_TOKEN' \
  -H 'Content-Type: application/json' \
  -d '{
    "bot_id": "7384582958305058867",
    "user_id": "user_123456",
    "query": "你好，请介绍一下你自己",
    "stream": false
  }'
```

### 4.2 流式对话接口（Chat Stream）

用于获取流式响应的对话结果。

#### 4.2.1 接口地址

```
POST /v1/chat
```

#### 4.2.2 请求参数

与标准对话接口相同，但需设置 `stream: true`

#### 4.2.3 响应格式

流式响应使用 Server-Sent Events (SSE) 格式：

```
data: {"id":"chat_123","content":"你","type":"answer"}

data: {"id":"chat_123","content":"好","type":"answer"}

data: {"id":"chat_123","content":"！","type":"answer","status":"completed"}
```

#### 4.2.4 请求示例

```bash
curl -X POST 'https://api.coze.cn/v1/chat' \
  -H 'Authorization: Bearer YOUR_PAT_TOKEN' \
  -H 'Content-Type: application/json' \
  -d '{
    "bot_id": "7384582958305058867",
    "user_id": "user_123456",
    "query": "你好",
    "stream": true
  }'
```

### 4.3 获取 Bot 信息接口

#### 4.3.1 接口地址

```
GET /v1/bot/get_online_info
```

#### 4.3.2 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| bot_id | string | 是 | 智能体 ID |

#### 4.3.3 请求示例

```bash
curl -X GET 'https://api.coze.cn/v1/bot/get_online_info?bot_id=7384582958305058867' \
  -H 'Authorization: Bearer YOUR_PAT_TOKEN'
```

#### 4.4.1 接口地址

```

```

#### 4.4.2 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| workflow_id | string | 是 | 工作流 ID |
| parameters | object | 是 | 工作流输入参数 |
| bot_id | string | 否 | 关联的 Bot ID |

#### 4.4.3 请求示例

```bash
curl -X POST 'https://api.coze.cn/v1/workflow/run' \
  -H 'Authorization: Bearer YOUR_PAT_TOKEN' \

  }'
```

### 4.5 获取会话历史接口

#### 4.5.1 接口地址

```
GET /v1/conversation/messages
```

#### 4.5.2 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| conversation_id | string | 是 | 会话 ID |
| limit | integer | 否 | 返回消息数量限制，默认 20 |

#### 4.5.3 请求示例

```bash

  -H 'Authorization: Bearer YOUR_PAT_TOKEN'
```

## 6. 响应格式说明

### 6.1 标准响应格式

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "id": "chat_7384582958305058867",
    "conversation_id": "conv_7384582958305058867",
    "bot_id": "7384582958305058867",
    "content": "你好！我是智能助手，很高兴为您服务。",
    "type": "answer",
    "status": "completed",
    "created_at": 1709280000,
    "usage": {
      "token_count": 50,
      "output_tokens": 30,
      "input_tokens": 20
    }
  }
}
```

### 6.2 响应字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| code | integer | 响应状态码，0 表示成功 |
| msg | string | 响应消息 |
| data | object | 响应数据 |
| data.id | string | 对话记录 ID |
| data.conversation_id | string | 会话 ID，用于后续对话 |
| data.bot_id | string | 智能体 ID |
| data.content | string | 智能体回复内容 |
| data.type | string | 消息类型：answer/text/tool_output |
| data.status | string | 状态：completed/in_progress |
| data.created_at | integer | 创建时间戳 |
| data.usage | object | Token 使用统计 |

### 6.3 流式响应格式

```
data: {"id":"chat_123","conversation_id":"conv_123","content":"你","type":"answer"}

data: {"id":"chat_123","conversation_id":"conv_123","content":"好","type":"answer"}

data: {"id":"chat_123","conversation_id":"conv_123","content":"！","type":"answer","status":"completed"}

data: [DONE]
```

## 7. 错误码说明

### 7.1 常见错误码

| 错误码 | 说明 | 解决方案 |
|--------|------|----------|
| 0 | 成功 | - |
| 400 | 请求参数错误 | 检查请求参数格式和必填项 |
| 401 | 认证失败 | 检查 Token 是否有效 |
| 403 | 权限不足 | 检查 Bot 发布状态和权限 |
| 404 | 资源不存在 | 检查 bot_id 是否正确 |
| 429 | 请求频率超限 | 降低请求频率 |
| 500 | 服务器内部错误 | 联系技术支持 |
| 503 | 服务不可用 | 稍后重试 |

## 8. 完整调用示例

### 8.1 Python 示例

```python
import requests
import json

class CozeBotClient:
    def __init__(self, api_token: str, bot_id: str):
        self.api_token = api_token
        self.bot_id = bot_id
        self.base_url = "https://api.coze.cn"
        self.conversation_id = None
    
    def chat(self, query: str, user_id: str = "default_user") -> dict:
        """
        与智能体进行对话
        
        Args:
            query: 用户输入的问题
            user_id: 用户 ID
            
        Returns:
            对话响应
        """
        url = f"{self.base_url}/v1/chat"
        
        headers = {
            "Authorization": f"Bearer {self.api_token}",
            "Content-Type": "application/json"
        }
        
        payload = {
            "bot_id": self.bot_id,
            "user_id": user_id,
            "query": query,
            "stream": False
        }
        
        # 如果有会话 ID，则传入以保持上下文
        if self.conversation_id:
            payload["conversation_id"] = self.conversation_id
        
        response = requests.post(url, headers=headers, json=payload)
        result = response.json()
        
        # 保存会话 ID 用于后续对话
        if result.get("code") == 0 and result.get("data"):
            self.conversation_id = result["data"].get("conversation_id")
        
        return result
    
    def chat_stream(self, query: str, user_id: str = "default_user"):
        """
        流式对话
        
        Args:
            query: 用户输入的问题
            user_id: 用户 ID
            
        Yields:
            流式响应内容
        """
        url = f"{self.base_url}/v1/chat"
        
        headers = {
            "Authorization": f"Bearer {self.api_token}",
            "Content-Type": "application/json"
        }
        
        payload = {
            "bot_id": self.bot_id,
            "user_id": user_id,
            "query": query,
            "stream": True
        }
        
        response = requests.post(url, headers=headers, json=payload, stream=True)
        
        for line in response.iter_lines():
            if line:
                line = line.decode('utf-8')
                if line.startswith('data: '):
                    data = line[6:]
                    if data == '[DONE]':
                        break
                    yield json.loads(data)


# 使用示例
if __name__ == "__main__":
    # 配置信息
    API_TOKEN = "your_personal_access_token"
    BOT_ID = "7384582958305058867"
    
    # 创建客户端
    client = CozeBotClient(API_TOKEN, BOT_ID)
    
    # 单轮对话
    response = client.chat("你好，请介绍一下你自己")
    if response.get("code") == 0:
        print("智能体回复:", response["data"]["content"])
        print("会话 ID:", response["data"]["conversation_id"])
    
    # 多轮对话
    response2 = client.chat("你能帮我做什么？")
    if response2.get("code") == 0:
        print("智能体回复:", response2["data"]["content"])
    
    # 流式对话
    print("\n流式对话:")
    for chunk in client.chat_stream("请写一首关于春天的诗"):
        if chunk.get("content"):
            print(chunk["content"], end="", flush=True)
```

### 8.2 Node.js 示例

```javascript
const axios = require('axios');

class CozeBotClient {
  constructor(apiToken, botId) {
    this.apiToken = apiToken;
    this.botId = botId;
    this.baseUrl = 'https://api.coze.cn';
    this.conversationId = null;
  }

  async chat(query, userId = 'default_user') {
    const url = `${this.baseUrl}/v1/chat`;
    
    const headers = {
      'Authorization': `Bearer ${this.apiToken}`,
      'Content-Type': 'application/json'
    };
    
    const payload = {
      bot_id: this.botId,
      user_id: userId,
      query: query,
      stream: false
    };
    
    if (this.conversationId) {
      payload.conversation_id = this.conversationId;
    }
    
    const response = await axios.post(url, payload, { headers });
    const result = response.data;
    
    if (result.code === 0 && result.data) {
      this.conversationId = result.data.conversation_id;
    }
    
    return result;
  }

  async *chatStream(query, userId = 'default_user') {
    const url = `${this.baseUrl}/v1/chat`;
    
    const headers = {
      'Authorization': `Bearer ${this.apiToken}`,
      'Content-Type': 'application/json'
    };
    
    const payload = {
      bot_id: this.botId,
      user_id: userId,
      query: query,
      stream: true
    };
    
    const response = await axios.post(url, payload, { 
      headers,
      responseType: 'stream'
    });
    
    for await (const chunk of response.data) {
      const lines = chunk.toString().split('\n');
      for (const line of lines) {
        if (line.startsWith('data: ')) {
          const data = line.slice(6);
          if (data === '[DONE]') return;
          yield JSON.parse(data);
        }
      }
    }
  }
}

  if (response.code === 0) {
    console.log('智能体回复:', response.data.content);
    console.log('会话 ID:', response.data.conversation_id);
  }
  
  // 多轮对话

    console.log('智能体回复:', response2.data.content);

  // 流式对话

      process.stdout.write(chunk.content);

}
```

### 8.3 Java 示例

```java
import okhttp3.*;
import com.fasterxml.jackson.databind.*;

import java.io.IOException;

public class CozeBotClient {
    private final String apiToken;
    private final String botId;
    private final String baseUrl = "https://api.coze.cn";
    private final OkHttpClient client = new OkHttpClient();
    private final ObjectMapper objectMapper = new ObjectMapper();
    private String conversationId;

    public CozeBotClient(String apiToken, String botId) {
        this.apiToken = apiToken;
        this.botId = botId;
    }

    public String chat(String query, String userId) throws IOException {
        String url = baseUrl + "/v1/chat";
        
        ObjectNode payload = objectMapper.createObjectNode();
        payload.put("bot_id", botId);
        payload.put("user_id", userId);
        payload.put("query", query);
        payload.put("stream", false);
        
        if (conversationId != null) {
            payload.put("conversation_id", conversationId);
        }
        
        RequestBody body = RequestBody.create(
            payload.toString(), 
            MediaType.parse("application/json")
        );
        
        Request request = new Request.Builder()
            .url(url)
            .post(body)
            .addHeader("Authorization", "Bearer " + apiToken)
            .addHeader("Content-Type", "application/json")
            .build();
        
        try (Response response = client.newCall(request).execute()) {
            String responseBody = response.body().string();
            JsonNode result = objectMapper.readTree(responseBody);
            
            if (result.get("code").asInt() == 0 && result.has("data")) {
                conversationId = result.get("data").get("conversation_id").asText();
                return result.get("data").get("content").asText();
            }
            
            throw new IOException("API Error: " + result.get("msg").asText());
        }
    }

    public static void main(String[] args) {
        CozeBotClient client = new CozeBotClient(
            "your_personal_access_token",
            "7384582958305058867"
        );
        
        try {
            String response = client.chat("你好，请介绍一下你自己", "user_123");
            System.out.println("智能体回复: " + response);
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

## 9. 常见问题

### 9.1 如何获取 Bot ID？

1. 登录扣子控制台
2. 进入您的 Bot 页面
3. 在 Bot 详情页面或发布页面可以找到 Bot ID
4. Bot ID 通常是一串数字，如 `7384582958305058867`

### 9.2 如何实现多轮对话？

多轮对话需要保存和复用 `conversation_id`：

1. 首次对话时，响应中会返回 `conversation_id`
2. 后续对话时，在请求中携带该 `conversation_id`
3. 智能体会根据会话历史保持上下文连贯性

### 9.3 流式响应如何使用？

1. 设置请求参数 `stream: true`
2. 使用 SSE 协议解析响应
3. 持续接收 `data:` 开头的消息直到 `[DONE]`

### 9.4 Token 过期如何处理？

1. Token 有过期时间，请定期更新
2. 如 Token 失效，重新在控制台生成
3. 建议在代码中实现 Token 自动刷新机制

### 9.5 请求频率限制是多少？

- 具体频率限制取决于您的账号类型
- 如遇 429 错误，请降低请求频率
- 企业用户可联系官方提升限额

### 9.6 如何调试 API 调用？

1. 检查请求头是否正确设置
2. 验证 Token 是否有效
3. 确认 Bot ID 是否正确
4. 检查 Bot 是否已发布为 API
5. 查看错误码和错误信息
